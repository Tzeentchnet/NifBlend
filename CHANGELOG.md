### Added

- **Phase 8i — texture-path utilities tied to per-game `Data/` roots**: pure resolution helpers in [`nifblend/bridge/textures.py`](nifblend/bridge/textures.py) (`normalize_nif_relative` strips a leading `data/` prefix + normalises separators; `resolve_texture_path(nif_relative, *, data_root, mode, extra_roots=())` implements all three `texture_resolution_mode` preferences — `STRICT` verbatim-exists, `CASE_INSENSITIVE` walks path components with per-level `listdir` + case-fold match, `FUZZY_LOOSEN_ROOT` tries `data_root` then each `extra_roots` entry in turn; every filesystem call is injectable for headless tests; `relative_to_data_root` case-insensitively re-expresses an absolute path relative to a configured root; `audit_image_paths` runs the resolution over an iterable of `(image_name, image_filepath)` pairs returning a list of `TextureAuditEntry` records). Operators in [`nifblend/ops/textures.py`](nifblend/ops/textures.py): [`NIFBLEND_OT_relink_textures_against_data_root`](nifblend/ops/textures.py) walks `bpy.data.images`, picks the active `GameProfile` (active-object stamp → prefs default → `UNKNOWN`), pulls the matching `Data/` root from `NIFBLEND_AddonPreferences`, optionally layers every other configured root as fallback when `FUZZY_LOOSEN_ROOT` is selected, and on hit updates `image.filepath`; misses are stashed on `scene.nifblend_texture_misses` (a `CollectionProperty` of `NIFBLEND_PG_texture_miss`) so the sidebar UIList can surface them. [`NIFBLEND_OT_bake_diffuse_to_texture`](nifblend/ops/textures.py) creates a fresh image, inserts an active `TEX_IMAGE` node in the material graph, and calls `bpy.ops.object.bake(type=...)` (pass selectable: `DIFFUSE` / `EMIT` / `NORMAL` / `ROUGHNESS`). New sidebar section in [`NIFBLEND_PT_utilities`](nifblend/ui/sidebar.py) surfaces both operators plus a [`NIFBLEND_UL_texture_misses`](nifblend/ops/textures.py) UIList that lights up only when the last relink reported misses. Test suite [`nifblend/tests/test_textures.py`](nifblend/tests/test_textures.py) (20 tests) covers `normalize_nif_relative` over 9 shapes (backslash / forward-slash / mixed case `data` prefix / leading separators / empty input), `resolve_texture_path` STRICT hit + case-sensitive miss, `CASE_INSENSITIVE` walking mixed-case components + missing-file miss, `FUZZY_LOOSEN_ROOT` first-hit-wins over multiple roots + empty-extras fallback, empty-input early-return, unknown-mode `ValueError`, leading-`Data\\` prefix stripping, `relative_to_data_root` hit / case-insensitive hit / outside-root miss / empty-input miss / equal-paths miss, `audit_image_paths` mixed relative + absolute + missing + blank entries returning `TextureAuditEntry` dataclass records. Operator-level coverage (live `bpy.data.images` walk, real Blender bake) stays deferred to Phase 8j. All 474 tests pass.

### Fixed

- Codegen `_emit_array_read` and the array branch of `_emit_field_write_body` now honour the schema's `arg=` attribute on array-of-struct fields by emitting `ctx.push_arg(...)` / `ctx.pop_arg()` around the loop. Previously only the single-struct read/write paths pushed the arg, so `BSTriShape.vertex_data` (an array of `BSVertexDataSSE` whose layout depends on `Vertex Desc >> 44`) decoded every vertex with `ctx.arg = 0` and produced all-zero records on read / wrote zero-length records on write. End-to-end BSTriShape round-trip now succeeds.
- Codegen-emitted bitfield classes now expose `__int__` / `__index__` / `__rshift__` / `__lshift__` / `__and__` / `__or__` (plus an internal `_packed()` helper). The schema models bitfield-typed fields as raw integers in `cond` / `arg` expressions (e.g. `Vertex Desc >> 44` for the SSE vertex flag word); without the dunders the emitted Python tried to shift the unpacked dataclass and raised `TypeError`.
- `nifblend/bridge/mesh_in._resolve_name` now correctly handles the codegen's `string` Compound (`.index` u32 for >= 20.1.0.3, inline `SizedString` for older files) instead of treating `block.name` as a raw int — the previous version coerced the dataclass into an int and fell through to the fallback path on every modern NIF.
- Codegen `_build_guard` now emits the closing parenthesis of any `False  # CODEGEN-TODO ...` fallback on its own line. Previously, when the schema's `cond` text itself contained a `#` (e.g. `!((Flags #BITAND# 1) != 0)`), the inline TODO comment swallowed the wrapper paren and produced a `SyntaxError` in `nifblend/format/generated/blocks.py` (`NiBlendInterpolator.read`). The fix only affects how guard expressions are line-broken; semantics of the emitted guard are unchanged.
- Codegen `cond_compiler` now accepts numeric literals with a leading `+` and signed-exponent floats (e.g. `+3.402823466e+38`, `-1.0E-7`) — the operand regex was previously `-?` only, so positive-signed exponent constants raised `ValueError: unexpected operand`.
- Codegen `_class_name` collapses C++-style `::` scope separators (`BSSkin::Instance` → `BSSkinInstance`) so Bethesda namespaced niobjects emit valid Python class names.
- `pytest.ini`'s `testpaths` now also includes `tools/codegen/tests` so the cond-compiler suite runs under the default `pytest` invocation (was previously only collected when explicitly pointed at).
